# Phoenix Manager — Marco 3.5 Entity Patches (Design)

**Date:** 2026-07-20  
**Status:** Approved  
**Depends on:** [Marco 3 Saves & Mods](2026-07-20-marco-3-saves-mods-design.md)

## Intent

Extend saves with **entity-level patches** (club reputation changes) while keeping core+mods as baseline. Demo mutation: match winners gain +1 reputation (capped at 100).

## Save format v2

```json
{
  "version": 2,
  "…runtime fields…",
  "patches": {
    "clubs": [{ "id": "london-fc-en", "changes": { "reputation": 83 } }]
  }
}
```

Load: `loadWorld` → `applyClubPatches` → restore table/matchday.

## Runtime

- Keep `baselineClubs` Map after world load.
- After each match in `advanceDay`, if home wins / away wins → bump winner reputation.
- On save: `diffClubs(baseline, world.clubs)` → `patches.clubs`.

## UI

Table column **Rep** showing current club reputation.

## Out of scope

players.patch mutations, separate `.patch` files on disk, transfers.
